# 핀볼 룰렛 — QA 인수 테스트 (Acceptance Tests)

대상: `/` (Pinball Roulette 단일 페이지). 환경: `pnpm dev` → `http://localhost:3000`.
실행 도구: Chrome DevTools MCP(`mcp__chrome-devtools__*`) + 수동 브라우저 검증.

각 케이스는 **준비 → 실행 → 통과 조건** 구조. 콘솔 에러는 **모든 케이스 공통 실패 조건**.

## TC-10. 🛑 맵 뒤집기 (`mapFlip`) — 핵심 인수

| 준비 | 4명 등록, 맵: `Wheel of Fortune` 선택, 특수효과: `맵 뒤집기` 활성, 시작 |
| 실행 | 진행 2/3 지점에서 자동 발동 → (1) 흔들림 0.6s (2) 회전 1.4s (3) 장애물 미러링 (4) 하향 임펄스로 재개 |
| **통과 (필수)** | **A. 양옆 바운더리 보존** — 좌·우 수직벽이 회전 후에도 그대로 유지되어야 한다 (마블 x좌표가 `[wallLeftX + r, wallRightX - r]` 범위 내). |
| | **B. x축 이탈 0건** — 모든 마블의 x좌표가 양옆 벽 안쪽에 머무름. 캔버스 좌/우 검은 영역으로 마블이 빠져나가는 프레임 없음. |
| | **C. GOAL 도달 보장** — 마블이 장애물에 끼어 영원히 정체되지 않음. 정체 시 자동 임펄스/넛지로 해소되어 finite 시간 내 모두 골인. |
| | D. 벽/바닥 색상이 미러링 후에도 깨지지 않음 (시각적 회귀 없음). |
| | E. 모든 5개 맵에 대해 위 조건이 동일하게 성립해야 함. |

## TC-11. 🛑 중력 거스리기 (`gravityReverse`) — 핵심 인수

| 준비 | 4명 등록, 맵: 각 맵 순회, 특수효과: `중력 거스리기` 활성, 시작 |
| 실행 | 진행 2/3 지점에서 자동 발동 → 중력 반전, 슬로프/각속도 반전, goalY가 spawn 위로 이동, 모든 마블에 위 임펄스 |
| **통과 (필수)** | **A. 양옆 바운더리 보존** — 좌·우 수직벽이 그대로 유지. |
| | **B. x축 이탈 0건** — 마블이 양옆 벽을 뚫고 나가지 않음. (반전 슬로프 + 임펄스에도 x축 범위 내) |
| | **C. GOAL 도달 보장** — 모든 마블이 finite 시간 내 새 goalY(상단 부근)에 도달. 장애물 끼임 발생 시 임펄스/넛지로 해소. |
| | D. ambient 사운드 시작/종료 정상 (`startAmbientLoop` / `stopAmbientLoop`). |
| | E. 보텍스 파티클이 `MAX_PARTICLES` 상한 초과로 누적되지 않음 (FPS 안정). |

## Chrome DevTools MCP 실행 시나리오 (자동)

다음 순서로 실행:

1. `new_page` → `http://localhost:3000`
2. `take_snapshot` (DOM 트리 + uid 확보)
3. `take_screenshot` — 초기 화면
4. `fill` 텍스트영역 → `홍길동, 김철수, 이영희, 박민수` 4명
5. `click` "참가자 등록" 버튼
6. `select` 맵 드롭다운 → 각 맵 순회
7. **TC-10 시퀀스 (각 맵)**: 특수효과 `맵 뒤집기` 클릭 → 시작 → 5초 wait → 스크린샷 → 게임 종료까지 wait → 스크린샷 → x축 이탈 검증(`evaluate_script`로 마블 좌표 추출)
8. **TC-11 시퀀스 (각 맵)**: 특수효과 `중력 거스리기` 클릭 → 시작 → 시퀀스 동일
9. `list_console_messages` — 콘솔 에러 0건 확인
10. `list_network_requests` — 4xx/5xx 0건 확인
